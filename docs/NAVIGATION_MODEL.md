# Navigation Model

## Главный принцип

`diez-control-center` — единый головной центр управления.

Навигация должна быть понятной для сотрудников и зависеть от роли пользователя.

## Основные разделы

1. Главная панель
2. Заказы
3. Чат / Сообщения
4. Ozon
5. Клиенты
6. Расчёты
7. Производство
8. Оплаты
9. Доставка
10. Товары
11. Настройки

## Настройки

Внутри настроек:

* Материалы и закупочные цены;
* Пользователи и роли;
* Интеграции;
* Способы оплаты;
* Способы доставки;
* Справочники;
* Системные настройки.

## Важно

Текущий экран "Материалы" временно находится на первом месте только для MVP-проверки.

В будущем он должен быть перенесён в:

```text
Настройки -> Материалы и закупочные цены
```

Главный рабочий экран должен быть:

```text
Заказы
```

## Роли и видимость разделов

Точная матрица прав должна быть спроектирована позже вместе с авторизацией, пользователями, audit log и моделью действий.

Ориентировочная видимость разделов:

| Роль | Видимость |
| --- | --- |
| admin | Все разделы, все настройки, пользователи и роли, интеграции, материалы и закупочные цены. |
| manager | Главная панель, Заказы, Чат / Сообщения, Ozon, Клиенты, Расчёты, Доставка, Товары в рабочем режиме. Без случайного изменения закупочных цен. |
| designer | Главная панель, связанные заказы, сообщения по макетам, задачи по дизайну, файлы и производственные комментарии. |
| production | Главная панель, заказы в производстве, Производство, производственные статусы, материалы в режиме просмотра при необходимости. |
| accountant | Главная панель, заказы в части оплат и документов, Клиенты/реквизиты, Оплаты, счета и закрывающие документы. |
| viewer | Только просмотр разрешённых разделов без изменения данных. |

## Не делать сейчас

* не реализовывать роли в коде;
* не менять текущую навигацию в UI;
* не создавать таблицы;
* не создавать миграции.
